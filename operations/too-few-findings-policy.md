# Too Few Findings Policy

Use this rule when the site looks better than expected or the findings are too weak for a useful paid report.

## Why this exists

The fastest way to damage this offer is to sell a paid report that says almost nothing.

If the audit is weak, the answer is not to add filler. The answer is to stop or narrow honestly.

## Minimum threshold for a paid mini-audit

A paid report should usually contain at least one of these:

- `2` practical issues with real business relevance
- `1` medium-priority issue plus `2` smaller useful improvements
- a clear enough action plan that a webmaster can immediately act on it

If none of that is true, pause before delivery.

## When to stop and not force the sale

Stop or downgrade the job if:

- findings are only cosmetic and trivial
- there is no meaningful trust, mobile, SEO, speed, privacy/cookie, or contact-path material
- the report would mostly repeat generic best practices with no site-specific value

## What to do instead

### Option 1. Keep it free

Send a short honest mini-analysis and do not push the paid report.

Use this when there is not enough real material.

### Option 2. Re-scope

If the client still wants help, offer a narrower next step only if justified, for example:

- homepage copy clarity review
- CTA/contact-path review
- webmaster-ready change list

Do not invent a deeper security package just to save the sale.

## Recommended message

"Ho guardato il sito e, onestamente, non vedo abbastanza materiale utile per un report completo a pagamento da €49. Posso lasciarti una mini-sintesi gratuita con i 2-3 punti piu utili emersi, cosi hai comunque qualcosa di concreto senza forzare il lavoro."

## Logging rule

If a paid audit is paused because of weak findings, record it in:

- `clients/<cliente>/handoff.md`
- `operations/delivery-log.csv`
